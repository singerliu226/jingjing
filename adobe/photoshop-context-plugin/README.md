# 菁菁小画桌 · Photoshop 只读现场

这是一个 Photoshop UXP 面板原型。它只读取当前文档的名称、尺寸、色彩模式、图层树、活动图层和可用的选区边界，并把结构化事件发给小画桌的 Context Bridge。设计师还可以在手动发送前勾选最长边 960px 的合成画布预览。

它不会修改文档，不会自动保存，不会上传 PSD 原文件，也不会在未经确认时把事件归档为项目版本。自动监听事件只发送元数据；低清预览必须由设计师手动勾选并发送。

## 本地加载

1. 安装 Adobe UXP Developer Tool。
2. 选择 `Add Plugin`，载入本目录的 `manifest.json`。
3. 启动小画桌本地服务。
4. 在面板中填写小画桌当前项目 ID、Bridge 地址和 `DESIGN_DESK_API_TOKEN`。
5. 打开一个文档，点击“发送当前上下文”，再回到小画桌确认候选。

## macOS 网络限制

Adobe UXP 在 macOS 上通常不允许插件访问明文 HTTP 地址。当前面板保留了 `http://127.0.0.1:4174` 供允许明文本地网络的开发环境使用；在 macOS Photoshop 中应为 Bridge 配置可信的本地 HTTPS 入口，并改用 `https://127.0.0.1:4175` 或对应地址。

不要把阿里云 DashScope 密钥填进插件。插件只使用独立的 `DESIGN_DESK_API_TOKEN`。

## 当前兼容矩阵

| 环境 | 能力 | 降级策略 |
| --- | --- | --- |
| Photoshop 23.3+ / Manifest v5 | 文档、图层、活动图层、事件与手动低清画布预览 | 选区边界不可用时只发送活动图层 |
| Photoshop 23.3 以下 | 不加载本插件 | 继续使用小画桌的授权文件夹或手动图片流程 |
| macOS + 无可信 HTTPS Bridge | 面板可以读取本地上下文，但网络发送可能被 UXP 拒绝 | 配置可信本地 HTTPS；在此之前使用授权文件夹 |
| 预览编码失败 | 文档与图层元数据仍可发送 | 面板会明确提示“仅发送上下文” |

`open`、`save`、`saveAs`、`close`、`select` 事件会触发只读元数据发送。图层树最多发送 200 项，嵌套最多 8 层。
