# 小画桌 Context-first Agent 路线与 To-dos

## 北极星

把小画桌从“等待设计师描述问题的聊天助手”，升级为“能在授权范围内理解设计现场、作品变化和选择过程的设计伙伴”。

语言不会消失，但只负责补充工具无法观察到的信息：业务目标、利益关系、截止时间和主观意图。画布、图层、选区、版本变化与偏好信号由 Agent 自己获取。

## 不可破坏的原则

- 默认只读：没有明确确认，不修改原稿。
- 最小采集：不做持续全屏录制，只采集用户授权的文档、选区和关键事件。
- 可见可撤销：采集状态、建议依据和写回操作必须对用户可见。
- 保留视觉信息：图片、结构和操作轨迹不强制先翻译成自然语言。
- 证据化偏好：审美判断必须能追溯到“选择了什么、拒绝了什么”，不能假装读心。
- 渐进接入：先验证一条闭环，再扩展 Photoshop、Illustrator、Figma 和生成工具。

## 交付路线总览

### M0｜方向、边界与数据地基

- [x] `CTX-001` 将产品方向从 Chat-first 调整为 Context-first。
- [x] `CTX-002` 定义采集、建议、写回三层权限原则。
- [x] `CTX-003` 把整条路线拆为可追踪 To-dos。
- [x] `CTX-004` 定义设计版本、上下文事件、偏好信号的首版数据结构。
- [x] `CTX-005` 为旧状态增加无损迁移，避免已有项目数据失效。
- [x] `CTX-006` 增加用户可见的“当前采集范围”设置。
- [ ] `CTX-007` 增加项目级数据导出与彻底删除。

验收：旧项目可以正常打开；新增 Context-first 字段不会把图片正文塞进浏览器持久化；写回策略默认为确认后执行。

### M1｜手动图片闭环：版本 → 判断 → 偏好

- [x] `CTX-101` 上传设计图时自动登记为项目设计版本。
- [x] `CTX-102` 将版本事件写入项目上下文时间线。
- [x] `CTX-103` 在图片旁显示版本编号。
- [x] `CTX-104` 提供“保留 / 更接近 / 方向不对”轻反馈。
- [x] `CTX-105` 将轻反馈保存为偏好信号，而不是普通聊天文字。
- [x] `CTX-105A` 支持直接粘贴截图进入版本流程，绕过文件选择和语言描述。
- [x] `CTX-106` 支持在画面上圈选区域并记录归一化坐标。
- [x] `CTX-107` 将圈选区域与版本、反馈、模型建议关联。
- [x] `CTX-107A` 将图片预览保存在浏览器本地 IndexedDB，刷新后仍可继续圈选与比较。
- [x] `CTX-108` 提供两个版本的并排对比视图。
- [x] `CTX-109` 让模型先描述可见变化，再解释可能原因，避免反向编造。
- [x] `CTX-110` 增加“这几张是方案 A/B，不是连续版本”的批次标记。

验收：设计师上传两版图片后，不写长提示词也能获得基于版本的复评；一次点击即可留下偏好证据。

### M2｜本地工作现场桥接

- [x] `CTX-201` 设计本地 Bridge 协议与鉴权方式。
- [x] `CTX-202` 监听用户主动授权的项目文件夹。
- [x] `CTX-203` 识别新增、保存、重命名和导出事件。
- [x] `CTX-204` 对文件生成内容指纹，避免重复记录同一版本。
- [x] `CTX-205` 只生成低分辨率预览和必要元数据。
- [x] `CTX-206` 在小画桌显示“刚刚检测到一个新版本”，由用户确认归档。
- [x] `CTX-207` 支持暂停监听、撤销授权和清理本地缓存。
- [x] `CTX-208` 建立离线队列，网络恢复后由用户决定是否同步。

验收：保存一张设计稿后，小画桌能在不要求重新上传的情况下发现候选版本；未授权目录不会被读取。

### M3｜Photoshop 只读插件

- [x] `CTX-301` 创建 Photoshop UXP 插件工程。
- [x] `CTX-302` 读取当前文档 ID、名称、尺寸和色彩模式。
- [x] `CTX-303` 读取图层树、图层类型、可见性和基础样式。
- [x] `CTX-304` 读取当前选区和活动图层。
- [x] `CTX-305` 手动确认后导出低分辨率合成画布预览。
- [x] `CTX-306` 捕获打开、保存、切换文档等关键事件。
- [x] `CTX-307` 将插件事件发送到本地 Bridge。
- [x] `CTX-308` 在插件内显示采集指示、暂停和发送按钮。
- [x] `CTX-309` 建立 Photoshop 版本兼容矩阵和降级策略。

验收：选中一个图层后，Agent 能说清“正在讨论哪个对象”，并给出与该对象相关的建议；插件不能修改文档。

### M4｜情境建议与打扰控制

- [ ] `CTX-401` 建立“是否值得打扰”的决策器。
- [ ] `CTX-402` 区分主动求助、重要风险和普通操作。
- [ ] `CTX-403` 建立建议冷却时间与重复建议去重。
- [ ] `CTX-404` 每条建议显示依据：画布、选区、版本差异或项目约束。
- [ ] `CTX-405` 支持“稍后提醒 / 这类不要提醒 / 只在交付前提醒”。
- [ ] `CTX-406` 记录建议是否被采纳及后续版本是否改善。
- [ ] `CTX-407` 建立错误归因反馈：“你看错对象了 / 目标理解错了 / 建议太早”。

验收：Agent 不会因为每次小改动都弹出建议；用户能知道它为什么此刻出现。

### M5｜安全写回与方案分支

- [ ] `CTX-501` 定义可写动作白名单。
- [ ] `CTX-502` 所有写回先生成操作预览。
- [ ] `CTX-503` 默认创建副本或新图层，不覆盖原稿。
- [ ] `CTX-504` 写回前显示影响对象和参数。
- [ ] `CTX-505` 写回后保存前后快照和撤销入口。
- [ ] `CTX-506` 支持生成配色、字体层级和版式实验分支。
- [ ] `CTX-507` 将分支选择记录为偏好信号。
- [ ] `CTX-508` 高风险操作要求二次确认。

验收：任何自动修改都能预览、确认、撤销并追溯；原始设计文件始终可恢复。

### M6｜可解释的审美与工作习惯记忆

- [ ] `CTX-601` 分离个人偏好、品牌规范和单项目方向。
- [ ] `CTX-602` 通过 A/B、保留、拒绝和撤销建立偏好样本。
- [ ] `CTX-603` 对偏好设置最小证据数和置信度。

- [ ] `CTX-605` 每条偏好可查看证据、纠正和删除。
- [ ] `CTX-606` 检测偏好漂移，避免永久固化旧风格。
- [ ] `CTX-607` 禁止把单次客户要求误记为设计师个人偏好。
- [ ] `CTX-608` 用后续真实选择评估偏好预测，而不是只看模型自评。

验收：Agent 的偏好推断有证据、有置信度、可纠正；不会用一句“你喜欢高级感”概括设计师。

## 当前迭代

`M1`、`M2` 与 `M3` 已完成。

本轮交付：

1. 统一 `context-event-v1` 协议、独立本地鉴权和持久队列。
2. 浏览器授权目录监听、SHA-256 指纹和候选版本确认。
3. 暂停、断开清理与需手动确认的离线同步。
4. Photoshop UXP 只读插件：文档、图层树、活动图层、选区和关键事件。
5. Photoshop 手动发送时可附带低分辨率合成画布预览；自动事件仍只发元数据。

下一轮进入 `M4 / CTX-401`：建议时机与打扰控制。

## Context Bridge v1

- 入口：`/api/context-bridge/status`、`/events`、`/events/:id/action`、`/control`。
- 鉴权：优先使用独立的 `DESIGN_DESK_API_TOKEN`；未配置时只接受回环地址请求。
- 来源：`browser_folder`、`photoshop`、`illustrator`、`figma`、`generated_tool`。
- 状态：`pending`、`imported`、`dismissed`。
- 去重：适配器提供稳定 `dedupeKey`，服务端同时保留事件指纹并限制队列长度。
- 图片边界：浏览器只生成最长边 960px 的 JPEG 预览；PSD、AI、PDF 默认只传元数据。
- 写回边界：Bridge v1 没有任何修改设计文档的接口。

## 首版数据契约

### 设计版本

```json
{
  "id": "v_xxx",
  "name": "V2",
  "projectId": "p_xxx",
  "createdAt": "ISO-8601",
  "source": "manual_upload",
  "artifact": {
    "attachmentId": "att_xxx",
    "fileName": "poster-v2.png",
    "mimeType": "image/png",
    "size": 120034,
    "width": 1080,
    "height": 1440,
    "fingerprint": ""
  }
}
```

### 上下文事件

```json
{
  "id": "ctx_xxx",
  "type": "design_version_added",
  "projectId": "p_xxx",
  "versionId": "v_xxx",
  "source": "manual_upload",
  "createdAt": "ISO-8601",
  "payload": {}
}
```

### 偏好信号

```json
{
  "id": "pref_xxx",
  "projectId": "p_xxx",
  "versionId": "v_xxx",
  "signal": "keep",
  "source": "explicit_quick_action",
  "createdAt": "ISO-8601",
  "note": ""
}
```

偏好信号是证据，不直接等于长期偏好。只有跨多个版本重复出现后，才能形成可向用户展示的偏好推断。

### 圈选区域

```json
{
  "id": "region_xxx",
  "projectId": "p_xxx",
  "versionId": "v_xxx",
  "label": "区域 1",
  "x": 0.12,
  "y": 0.2,
  "width": 0.42,
  "height": 0.3,
  "source": "manual_drag",
  "createdAt": "ISO-8601"
}
```

区域坐标相对原图归一化：左上角为 `(0,0)`，右下角为 `(1,1)`，因此同一个圈选能跨预览尺寸和设备复用。
